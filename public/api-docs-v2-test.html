API Documentation

The Verity API provides programmatic access to our fact-checking infrastructure. Verify claims, batch process documents, and integrate verification into your workflow.

Base URL: https://veritysystems-production.up.railway.app

Authentication

All API requests require an API key. Include your key in the request header:

HTTP Header 
Authorization: Bearer YOUR_API_KEY

Keep your API key secure. Never expose it in client-side code or public repositories.

Quick Start

Here's a simple example to verify your first claim:

cURL 
curl -X POST https://veritysystems-production.up.railway.app/v3/verify \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"claim": "The Earth is approximately 4.5 billion years old"}'

Verify Claim

POST /v3/verify

Verify a single claim and receive a verdict with confidence score and sources.

Request Body

Parameter Type Description
claimrequired string The claim to verify (max 1000 chars)
deep_research boolean Enable deep research mode (default: false)
sources string[] Limit to specific sources

Response

{
  "verdict": "true",
  "confidence": 0.987,
  "explanation": "This claim is supported by extensive scientific evidence...",
  "sources": [
    {
      "name": "Wikipedia",
      "url": "https://en.wikipedia.org/wiki/Age_of_Earth",
      "type": "encyclopedia"
    }
  ],
  "models_consulted": ["gpt-4", "claude-3", "gemini-pro"],
  "timestamp": "2026-01-09T12:00:00Z"
}

Batch Verify

POST /v3/batch

Verify multiple claims in a single request. Maximum 100 claims per batch.

Request Body 
{
  "claims": [
    "The Great Wall of China is visible from space",
    "Humans only use 10% of their brain",
    "Goldfish have a 3-second memory"
  ]
}

Health Check

GET /health

Check API status and availability. No authentication required.

Error Codes

Code Description
400 Bad Request — Invalid parameters
401 Unauthorized — Invalid or missing API key
429 Too Many Requests — Rate limit exceeded
500 Internal Server Error — Something went wrong

Python SDK

Python 
import requests

def verify_claim(claim, api_key):
    response = requests.post(
        "https://veritysystems-production.up.railway.app/v3/verify",
        headers={"Authorization": f"Bearer {api_key}"},
        json={"claim": claim}
    )
    return response.json()

# Example usage
result = verify_claim("The Earth is round", "your-api-key")
print(result["verdict"])  # "true"

Node.js SDK

JavaScript 
async function verifyClaim(claim, apiKey) {
  const response = await fetch(
    'https://veritysystems-production.up.railway.app/v3/verify',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${apiKey}`
      },
      body: JSON.stringify({ claim })
    }
  );
  return response.json();
}

// Example usage
const result = await verifyClaim('The Earth is round', 'your-api-key');
console.log(result.verdict); // "true"
